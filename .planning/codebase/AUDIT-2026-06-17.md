## Test coverage

322 tests, **38% stmt / 26% branch** overall. `src/auth` (94%) and `src/ratelimit` (96%) are strong; `src/http/server.ts` (28%) and tools are weak.

- **8 tools have ZERO unit tests:** assess-platform-dependency, find-closest-competitor, find-why-now-signals, find-yc-rfs-alignment, get-category-failure-modes, map-competitive-weaknesses, read-competitor-changelog, scan-producthunt-launches.
- `_envelope-shape.test.ts` is **static-text only** (greps for a description clause) ‚Äî it invokes nothing.
- **`honest_gap` status is never behaviorally tested** ‚Äî 7 tools emit it; a tool could return `status:'ok'` with empty data and every test stays green. This guards the product's core anti-bias promise.
- `structural-validator` + `schema` have no dedicated vitest (only ride `finalize`/`report-schema` integration tests + tsx self-checks that don't run in `vitest run`).

**Top 5 missing tests (by value):** (1) `honest_gap` behavioral contract across the 7 tools; (2) dedicated `structural-validator` vitest (port the tsx self-check into CI); (3) `verdict-validator` per-gate source rules H4/H4b/H4c; (4) Gate-3 veto √ó H4b ordering edge (Gate 3 C/D-only ‚Üí INCONCLUSIVE before veto reads it); (5) `error` envelope on upstream API failure for 2-3 tools.
