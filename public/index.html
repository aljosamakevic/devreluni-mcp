Mechanism

How it works.

  1. G1 Direct Competitor Scan

    Who is the closest existing thing, what have they shipped, where are they weak?

  2. G2 Market Structure

    Is the market shaped so this idea can win meaningful share?

  3. G3 Platform Risk

    Will a platform shift or hyperscaler ship this in 24 months?

  4. G4 Willingness to Pay

    Will the target customer actually pay enough to make this a business?

  5. G5 Why Now

    What changed in the last 24 months that makes this possible now?

Verdict math

The fail-2 rule.

Five gate verdicts collapse to one overall verdict by a fixed table. No model discretion at the rollup.

Gate verdicts Overall
5 PASS GO
4 PASS + 1 INCONCLUSIVE CONDITIONAL GO
4 PASS + 1 FAIL CONDITIONAL GO
3 PASS + 2 (FAIL or INCONCLUSIVE) CONDITIONAL GO
2+ FAIL NO-GO

Anti-bias by construction

Four structural defenses.

Confirmation bias is the failure mode of every idea-validation tool. Veto answers it at four points in the pipeline.

  1. Every fact carries a quality tier (S/A/B/C/D) and a bias flag. No fact ships without provenance.
  2. Contradicting-evidence search is mandatory. Every gate fans out a counter-search before the verdict resolves.
  3. Three Validation Checks audit verdicts before they render. Internal consistency, source quality, anti-confirmation.
  4. The Spiky POV section is blank. The framework never decides for the user. The reader writes the take.

Tier 1 — Works today (bearer token)

CLI-native MCP clients.

Anthropic's native desktop app. The reference MCP host.

macOS: ~/Library/Application Support/Claude/claude_desktop_config.json

Windows: %APPDATA%\Claude\claude_desktop_config.json

https://getvetoed.com/mcp
{ "mcpServers": { "veto": { "url": "https://getvetoed.com/mcp", "headers": { "Authorization": "Bearer pv_<your_token>" } } } }

Tier 2 — Developer integration

OpenAI Responses API.

Pass Veto into a Responses API call as an MCP tool. For developers building agents on top of OpenAI's API — not a consumer ChatGPT install.

{ "type": "mcp", "server_label": "veto", "server_url": "https://getvetoed.com/mcp", "headers": { "Authorization": "Bearer pv_<your_token>" }, "require_approval": "never" }
